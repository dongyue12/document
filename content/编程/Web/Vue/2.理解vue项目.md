### 目录结构
```
my-vue-project/
├── node_modules/          # 项目依赖包（自动生成，不要手动修改）
├── public/                # 静态资源目录（会被直接复制，不经过构建）
│   ├── favicon.ico        # 网站图标
│   └── index.html         # 应用的 HTML 模板
├── src/                   # 源代码目录（核心开发区域）
│   ├── assets/            # 需要经过构建工具处理的资源（图片、样式、字体等）
│   ├── components/        # 可复用的 Vue 组件（如 Button.vue、Header.vue）
│   ├── views/             # 页面级别的组件（通常配合路由使用）
│   ├── router/            # Vue Router 配置（index.js）
│   ├── store/             # Vuex 或 Pinia 状态管理配置（index.js / modules）
│   ├── utils/             # 工具函数（可选，自己创建）
│   ├── api/               # 接口请求封装（可选）
│   ├── App.vue            # 根组件，整个应用的入口组件
│   └── main.js            # 应用的入口文件，初始化 Vue 实例、挂载路由/状态管理等
├── .gitignore             # Git 忽略文件配置
├── index.html             # （Vite 项目）入口 HTML 文件
├── package.json           # 项目依赖、脚本命令、元信息
├── package-lock.json / yarn.lock   # 依赖版本锁定文件
├── vite.config.js         # Vite 配置文件（如果用 Vite）
├── vue.config.js          # Vue CLI 配置文件（如果用 Vue CLI）
├── babel.config.js        # Babel 编译配置（Vue CLI 项目常见）
├── .eslintrc.js           # ESLint 代码规范配置
└── README.md              # 项目说明文档
```

### 总结

- 一个vue项目都是在index.html这一个单页面形成各种交互，这个就是所谓的SPA(Single Page Application)

- vue3的核心就是通过createApp的函数创建一个应用实例，在这个实例中构建各种应用(main.ts)
- 每个vue文件就是一个页面上的组件，组件可以嵌套使用
