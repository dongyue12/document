## 事件传参

### 1. 基本传参与访问事件对象

在方法中可以通过参数接收传入的值，也可以使用 `$event` 获取原生 DOM 事件对象。

```vue
<template>
  <h3>方法</h3>
  <div>现在的值是：{{ number }}</div>
  <button @click="add">add</button>
  <button @click="sub('减少')">sub</button>
</template>

<script>
export default {
  data() {
    return {
      number: 0
    }
  },
  methods: {
    add(e) {
      this.number++
      e.target.innerHTML = 'add' + this.number
      alert(msg)   // 注意：这里 msg 未定义，笔记中原样保留
    },
    sub(msg) {
      this.number--
      alert(msg)
    }
  }
}
</script>
```

> **说明**：  
> - 方法 `add(e)` 中的 `e` 是原生事件对象，可用于修改触发事件的 DOM 元素。  
> - 方法 `sub(msg)` 中的 `msg` 是从模板传入的参数 `'减少'`。

---

### 2. 通过事件获取列表中的值

使用 `v-for` 遍历列表，将当前项的值作为参数传递给方法。

```vue
<template>
  <h3>点击列表值</h3>
  <div @click="showName(value)" v-for="(value, index) in names" :key="index">
    {{ value }}
  </div>
</template>

<script>
export default {
  data() {
    return {
      names: ['张三', '李四', '王五']
    }
  },
  methods: {
    showName(value) {
      alert(value)
    }
  }
}
</script>
```

> **说明**：`@click="showName(value)"` 将当前循环项的 `value`（即姓名）传给 `showName` 方法。

---

### 3. 同时传递参数和事件对象

在传参的同时，使用 `$event` 将原生事件对象也传递给方法。

```vue
<template>
  <h3>点击列表值</h3>
  <div @click="showName(value)" v-for="(value, index) in names" :key="index">
    {{ value }}
  </div>

  <div @click="showName(value, $event)" v-for="(value, index) in names" :key="index">
    {{ value }}
  </div>
</template>

<script>
export default {
  data() {
    return {
      names: ['张三', '李四', '王五']
    }
  },
  methods: {
    showName(value, e) {
      alert(value)
      console.log(e)   // 打印原生事件对象
    }
  }
}
</script>
```

> **说明**：  
> - `$event` 是 Vue 提供的特殊变量，代表原生 DOM 事件对象。  
> - 方法中需要按顺序接收参数：先接收自定义参数（`value`），再接收事件对象（`e`）。

---

### 总结

| 场景                     | 模板写法                         | 方法定义                   |
| ------------------------ | -------------------------------- | -------------------------- |
| 只传自定义参数           | `@click="method(param)"`         | `method(param) { ... }`    |
| 只传事件对象（不传参数） | `@click="method"`                | `method(e) { ... }`        |
| 同时传参和事件对象       | `@click="method(param, $event)"` | `method(param, e) { ... }` |

## 事件修饰符

在处理事件时，经常需要调用 `event.preventDefault()` 或 `event.stopPropagation()`。虽然可以直接在方法内调用，但 Vue 为 `v-on` 提供了**事件修饰符**，让方法能更专注于数据逻辑，而不用去处理 DOM 事件的细节。

常用修饰符有：

- `.stop` – 阻止事件冒泡
- `.prevent` – 阻止默认行为
- `.once` – 事件只触发一次
- `.enter` – 按键修饰符（回车）

------

### 1. 阻止默认事件（.prevent）

#### 方式一：在方法中手动调用 `e.preventDefault()`

vue

```
<template>
  <h3>阻止默认事件</h3>
  <a @click="nota" href="https://www.bilibili.com/">跳转到B站</a>
</template>

<script>
export default {
  methods: {
    nota(e) {
      // 阻止默认事件（阻止链接跳转）
      e.preventDefault()
      alert('你还没有vip')
    }
  }
}
</script>
```



#### 方式二：使用 `.prevent` 修饰符（内联）

vue

```
<template>
  <h3>阻止默认事件</h3>
  <!-- 使用 .prevent 修饰符，无需在方法内写 e.preventDefault() -->
  <a @click.prevent="nota2" href="https://www.bilibili.com/">跳转到B站</a>
</template>

<script>
export default {
  methods: {
    nota2() {
      alert('你还没有vip')
    }
  }
}
</script>
```



> **说明**：
>
> - `.prevent` 修饰符等价于调用 `event.preventDefault()`
> - 使用修饰符后，方法可以只关注业务逻辑，不处理 DOM 事件细节

------

### 2. 阻止事件冒泡（.stop）

当点击子元素时，默认会触发父元素上的事件。使用 `.stop` 修饰符可以阻止冒泡。

vue

```
<template>
  <h3>阻止事件冒泡</h3>
  <div @click="divtest">
    <p @click="ptest">测试冒泡</p>
  </div>
</template>

<script>
export default {
  methods: {
    divtest() {
      console.log('div')
    },
    ptest(e) {
      // 阻止事件冒泡到父元素
      e.stopPropagation()
      console.log('p')
    }
  }
}
</script>
```



> **说明**：
>
> - 不使用 `.stop` 时，点击 `<p>` 会先输出 `'p'`，再输出 `'div'`
> - 使用 `e.stopPropagation()` 后，只会输出 `'p'`，父元素不会触发

------

### 修饰符总结

| 修饰符     | 作用                                 | 示例                      |
| :--------- | :----------------------------------- | :------------------------ |
| `.prevent` | 阻止默认行为（如链接跳转、表单提交） | `@click.prevent="method"` |
| `.stop`    | 阻止事件冒泡                         | `@click.stop="method"`    |
| `.once`    | 事件只触发一次                       | `@click.once="method"`    |
| `.enter`   | 按键修饰符，回车触发                 | `@keyup.enter="method"`   |
