## 计算属性

#### 1. 什么是计算属性？

计算属性用于 **声明式地描述依赖响应式数据的复杂逻辑**。  
它基于其依赖的响应式数据自动缓存，只有当依赖发生变化时才会重新计算。

#### 2. 为什么使用计算属性？

- **模板简洁**：将复杂逻辑从模板中抽离，避免模板臃肿。
- **自动缓存**：依赖不变时，多次访问直接返回缓存结果，性能更优。
- **响应式依赖追踪**：依赖的响应式数据变化时，计算属性自动更新。

#### 3. 计算属性 vs 方法

| 特性         | 计算属性 (`computed`)                  | 方法 (`methods`)                           |
| ------------ | -------------------------------------- | ------------------------------------------ |
| **缓存机制** | ✅ 依赖未变时返回缓存值，不重新执行     | ❌ 每次重新渲染都会执行函数                 |
| **触发更新** | 仅当依赖的响应式数据变化时重新计算     | 与渲染无关，每次调用都执行                 |
| **调用方式** | 当作属性使用，**不加括号** `{{ sum }}` | 当作方法调用，**必须加括号** `{{ sum() }}` |
| **适用场景** | 派生数据、复杂逻辑、需要缓存的计算结果 | 事件处理、非响应式操作                     |

### 示例代码

```vue
<template>
  <div>
    <h3>计算属性示例</h3>
    <!-- 显示人员列表 -->
    <ul>
      <li v-for="item in names" :key="item.name">
        {{ item.name }} - {{ item.age }}岁
      </li>
    </ul>

    <h3>计算属性结果（缓存）</h3>
    <p>内容是否非空：{{ hasContent }}</p>

    <h3>方法结果（每次渲染都执行）</h3>
    <p>内容是否非空：{{ checkContent() }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      names: [
        { name: "你", age: 19 },
        { name: "把", age: 15 }
      ],
      content: ["你", "把", "我", "他", "她"]
    };
  },
  computed: {
    // 计算属性：依赖 content 数组，仅当 content 变化时重新计算
    hasContent() {
      console.log('计算属性 hasContent 重新计算');
      return this.content.length > 0 ? 'yes' : 'no';
    }
  },
  methods: {
    // 普通方法：每次重新渲染都会被调用
    checkContent() {
      console.log('方法 checkContent 被执行');
      return this.content.length > 0 ? 'yes' : 'no';
    }
  }
};
</script>
```

#### 运行观察

- 打开控制台，修改 `content` 数组（例如通过按钮 push 新元素），会发现：
  - `hasContent` 计算属性仅在数组长度变化时打印一次日志。
  - 若模板中有其他地方重新渲染（如其他数据变化），`checkContent` 方法会再次执行并打印日志。

---

### 关键要点

1. **计算属性应返回一个值**，并在模板中作为属性直接使用（不加括号）。
2. **计算属性内不能有异步操作**（如 `setTimeout`、`fetch`），异步逻辑应使用 `methods` 或 `watch`。
3. **计算属性默认只有 `getter`**，需要时可提供 `setter` 实现双向绑定。
4. **避免计算属性与 `methods` 命名冲突**，否则 Vue 会报错或产生意外行为。
