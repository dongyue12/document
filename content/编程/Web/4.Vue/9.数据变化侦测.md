## 数据变化侦测

### Vue 响应式数组变更方法

Vue 能够侦听并响应以下数组方法的调用，从而自动触发视图更新：

| 变更方法    | 作用                         |
| ----------- | ---------------------------- |
| `push()`    | 向数组末尾添加一个或多个元素 |
| `pop()`     | 删除数组最后一个元素         |
| `shift()`   | 删除数组第一个元素           |
| `unshift()` | 向数组开头添加一个或多个元素 |
| `splice()`  | 添加/删除/替换数组元素       |
| `sort()`    | 对数组元素进行排序           |
| `reverse()` | 反转数组元素的顺序           |

> 这些方法会直接修改原数组，因此 Vue 能够拦截并触发更新。

---

### 非变更方法（不可变方法）

相对地，以下方法不会修改原数组，而是 **返回一个新数组**：

| 非变更方法 | 作用                               |
| ---------- | ---------------------------------- |
| `filter()` | 过滤数组元素，返回符合条件的新数组 |
| `concat()` | 合并多个数组，返回新数组           |
| `slice()`  | 截取数组的一部分，返回新数组       |

当使用非变更方法时，**必须将原数组替换为新数组**，才能触发 Vue 的响应式更新。例如：

```javascript
this.numbers1 = this.numbers1.concat(this.numbers2);
```

---

### 示例代码整合

以下是一个完整的 Vue 单文件组件示例，演示了变更方法与替换数组的应用场景：

```vue
<template>
  <div>
    <h3>数据变化侦测（变更方法）</h3>
    <ul>
      <li v-for="value in names" :key="value">{{ value }}</li>
    </ul>

    <h3>合并数据（非变更方法 + 替换数组）</h3>
    <button @click="merge">点击合并</button>

    <h3>数组1</h3>
    <p v-for="value in numbers1" :key="value">{{ value }}</p>

    <h3>数组2</h3>
    <p v-for="value in numbers2" :key="value">{{ value }}</p>
  </div>
</template>

<script>
export default {
  data() {
    return {
      names: ['geini', 'edan', 'fish'],
      numbers1: [1, 2, 3, 4, 5],
      numbers2: [4, 5, 6, 7, 8, 9, 10]
    };
  },
  methods: {
    merge() {
      // concat 不修改原数组，返回新数组，需通过赋值触发更新
      this.numbers1 = this.numbers1.concat(this.numbers2);
    }
  }
};
</script>
```

---

### 关键要点

1. **变更方法**：直接修改原数组，Vue 会自动侦测并更新 DOM。
2. **非变更方法**：返回新数组，必须通过 `this.xxx = newArray` 的形式重新赋值，才能触发响应式。
3. 在模板中使用 `v-for` 时，务必绑定唯一的 `key` 属性，以优化渲染性能。
